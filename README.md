# 雾灯旅馆

这是一个给 AI 盲玩的、单文件、零依赖、确定性的 Python 文字小游戏。主题是经营一间旅馆：每天处理房间、餐食、温泉、客诉和天气事件。目标不是最大收益，而是让旅馆留下记忆。

公开接口只有两个：

```python
from hotel_game import new_game, cmd

print(new_game("demo-seed"))
print(cmd("客人"))
print(cmd("去 客房; 安排 全部; 去 厨房; 做饭 全部"))
```

每次 `cmd(text)` 返回普通文字，并在最后附一行紧凑 JSON 状态栏。状态栏只含公开状态和待办数量，便于 AI 解析，不列出隐藏事件或未发现图鉴。

## 运行方式

无需安装依赖，直接导入：

```bash
python3 - <<'PY'
from hotel_game import new_game, cmd
print(new_game("my-seed"))
print(cmd("帮助"))
PY
```

也可以连续输入批量指令，分隔符支持分号、换行和“然后”：

```python
cmd("去 客房; 安排 全部; 去 温泉; 温泉 全部; 结束一天")
```

想让 AI 稳定推进主线时，也可以用快捷指令：

```python
cmd("照顾 全部")
```

它会按房间、客诉、餐食、温泉的顺序执行，仍然正常消耗体力、食材和柴火。如果今天客人多于房间数，游戏会提醒房间不够；餐食和温泉默认只服务已经安排房间的住客，避免把资源花在最后无法入住的候补客人身上。

AI 不知道下一步做什么时，可以问：

```python
cmd("建议")
```

`建议`、`今日计划`、`账本提醒` 只根据公开状态给提示，不会剧透隐藏事件或未发现图鉴。体力不够照顾全部时，它会给出优先级，而不是直接推荐标准接待。

## 收益 × Hospitality

雾灯旅馆不是收益最大化游戏，但新版加入了轻量的经营策略：每天可以先选择“今日定价”和“今日承诺”。

```python
cmd("定价 亲民")  # 亲民 / 标准 / 溢价
cmd("承诺 温泉")  # 均衡 / 安静 / 餐食 / 温泉 / 礼宾
cmd("收益")
```

- `亲民` 房费少一点，但客人更宽容。
- `标准` 价格和期待都居中。
- `溢价` 房费更高，但承诺落空时更容易退费。

承诺会在夜间结账时兑现或落空：例如承诺 `餐食` 就要把已入住客人的饭照顾好；承诺 `温泉` 就要留意想泡汤的人和柴火；承诺 `安静` 要处理客诉、检查风夜窗扣；承诺 `礼宾` 要通过招呼或客诉处理给客人抵达感。

年报会额外显示抽象版酒店指标：`OCC`、`ADR`、`RevPAR`，以及承诺兑现/落空次数。它们用于复盘经营倾向，不是唯一评分标准。

## 随机内容

随机内容由 seed 驱动，适合 AI 盲玩和复盘。同一个 seed 加同一串指令会得到同样结果。

- 每天会随机天气，并可能出现一个公开的“今日小事”，例如街价浮动、山路泥泞、泉眼升温等。
- 客人由职业和性格组合生成。职业决定愿望、报酬和可收集记忆；性格会影响餐食、温泉、打扫、招呼、客诉等行动的收益。
- 稀有事件按地点触发：厨房、温泉、客房、庭院、前厅和夜间都有不同的隐藏记忆。
- 最后一行 JSON 状态栏包含公开状态、时段、季节和今日事件标题，但不会列出未发现的隐藏图鉴。

## 时间、四季和年报

游戏内一年是 28 天，每季 7 天。每个消耗体力的行动会推进一天内的时段：清晨、上午、午后、傍晚、入夜、深夜。购物和升级也会花掉一点白天时间。

四季会改变天气权重和庭院生态。进入庭院时会看到当季植物和小动物，例如春天的新苔与燕子、夏天的绣球与萤火、秋天的红叶与蟋蟀、冬天的山茶与雪地足迹。

每天第一次进入庭院会出现一段庭院小记，并有机会获得“季节灵感”。灵感不会直接变钱，但会让当天某类照顾行动更温柔，例如做饭、招呼、安排房间、温泉或处理客诉时心情+1。

每年结束时，游戏会自动写一份年度总结：统计接待、收入、支出、疏漏、图鉴和记忆，并根据 AI 的行动偏好生成“AI 性格画像”。平时也可以输入 `年度总结`、`年报` 或 `性格` 查看当前年度观察。

## 存档

游戏会自动保存为 JSON。默认路径是同目录下的 `hotel_save.json`。

测试或多开时可以用环境变量指定存档位置：

```bash
HOTEL_GAME_SAVE=/tmp/my_hotel_save.json python3 your_player.py
```

也可以在游戏内使用命名存档和备份：

```python
cmd("保存为 一周目")
cmd("存档列表")
cmd("备份存档")
cmd("读档 一周目")
```

读档前会自动把当前进度放进备份，覆盖同名命名存档时也会备份旧版本。命名存档默认放在自动存档同目录的 `hotel_saves/` 下，备份放在 `hotel_saves/backups/` 下。

请不要让盲玩 AI 读取源代码里的内部表；让它只调用 `new_game(seed)` 和 `cmd(text)`。

## 记忆线

游戏有两类跨天记忆：

- 回头客：客人被照顾得足够好时，会被旅馆记住；以后可能再次来访，并认得柜台、钥匙牌或上次的房间。
- Staff：礼宾姐姐、厨房阿姨、接驳车司机、汤守、客房前辈会随着相关行动积累信任，逐步触发 staff 记忆。

## 常用指令

可说 `帮助` 查看不剧透的指令表。核心循环通常是：

- `客人` 查看今日住客
- `去 客房`、`去 厨房`、`去 温泉`、`去 庭院`、`去 商店` 切换地点
- `建议` 查看不剧透的今日计划
- `定价 溢价`、`承诺 温泉`、`收益` 调整和查看今日经营策略
- `安排 全部`、`做饭 1`、`温泉 全部`、`客诉 全部` 处理当天事务
- `照顾 全部` 按房间→客诉→餐食→温泉执行标准接待
- `备料`、`拾柴`、`打扫`、`招呼` 做资源或情绪行动
- `买 食材 3`、`升级 厨房` 使用商店
- `保存为 一周目`、`读档 一周目`、`存档列表`、`备份存档` 管理存档
- `图鉴` 查看已经发现的记忆
- `年度总结` 查看当前年度观察
- `结束一天` 结算并进入下一天

如果结束一天时还有客人没安排房间，结算文本会提醒：未安排房间的客人不会入住，哪怕已经吃过饭。

## Blind 版本

运行生成脚本可得到 `hotel_game_blind.py`：

```bash
python3 make_blind.py
```

生成版会保留 `new_game(seed)` 和 `cmd(text)`，同时把字符串字面量编码，减少源代码被直接阅读时的剧透。它不是安全加密，只是面向盲玩评测的防剧透包装。
